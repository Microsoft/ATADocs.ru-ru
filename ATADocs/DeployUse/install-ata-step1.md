## Шаг 1. Скачивание и установка центра ATA
Проверив соответствие сервера требованиям, можно приступить к установке центра ATA.

На сервере шлюза ATA сделайте следующее:

1.  Скачайте ATA из [центра оценки TechNet](http://www.microsoft.com/en-us/evalcenter/).

2.  Войдите под учетной записью пользователя, который является членом локальной группы администраторов.

3.  В командной строке, открытой с повышенными привилегиями, запустите файл Setup.EXE центра Microsoft ATA и следуйте указаниям мастера установки.

4.  На странице **приветствия** выберите язык и нажмите кнопку **Далее**.

5.  Прочтите лицензионное соглашение и, если вы принимаете условия, нажмите кнопку **Далее**.

6.  На странице **Конфигурация центра** введите следующие сведения с учетом используемой среды.

    |Поле|Описание|Комментарии|
    |---------|---------------|------------|
    |Путь установки|Это расположение для установки центра ATA. По умолчанию центр устанавливается в папку с путем %programfiles%\Microsoft Advanced Threat Analytics\Center.|Оставьте значение по умолчанию.|
    |Путь к данным базы данных|Это расположение, в котором будут размещены файлы базы данных MongoDB. По умолчанию используется путь %programfiles%\Microsoft Advanced Threat Analytics\Center\MongoDB\bin\data.|Выберите расположение, где предусмотрено достаточно пространства для будущего роста данных с учетом требований к размеру. **Примечание.** <ul><li>В рабочих средах следует использовать диск, на котором достаточно свободного места с учетом планирования ресурсов.</li><li>При крупных развертываниях база данных должна находиться на отдельном физическом диске.</li></ul>Дополнительные сведения о размерах см. в статье [Планирование производительности ATA](/advanced-threat-analytics/PlanDesign/ata-capacity-planning).|
    |Путь к журналу базы данных|Это расположение, в котором будут размещены файлы журнала базы данных. По умолчанию используется путь %programfiles%\Microsoft Advanced Threat Analytics\Center\MongoDB\bin\data\journal.|При крупных развертываниях журнал базы данных должен находиться на физическом диске, отличном от диска базы данных и системного диска. Выберите расположение, где достаточно места для журнала базы данных.|
    |IP-адрес службы центра ATA: порт|Это IP-адрес, который будет прослушивать служба центра ATA для взаимодействия через шлюзы ATA.<br /><br />**Порт по умолчанию:** 443|Нажмите клавишу СТРЕЛКА ВНИЗ, чтобы выбрать IP-адрес, который будет использовать служба центра ATA.<br /><br />IP-адрес и порт службы центра ATA не может совпадать с IP-адресом и портом консоли ATA. Измените порт консоли ATA.|
    |SSL-сертификат службы центра ATA|Это сертификат, который будет использовать служба центра ATA.|Щелкните значок ключа, что выбрать установленный сертификат или проверить самозаверяющий сертификат при развертывании в лабораторной среде.|
    |IP-адрес консоли ATA|Это IP-адрес, который будет использовать служба IIS для консоли ATA.|Нажмите клавишу СТРЕЛКА ВНИЗ, чтобы выбрать IP-адрес, который будет использовать консоль ATA. **Примечание**. Запишите этот IP-адрес, чтобы упростить доступ к консоли ATA через шлюз ATA.|
    |SSL-сертификат консоли ATA|Это сертификат, используемый службой IIS.|Щелкните значок ключа, что выбрать установленный сертификат или проверить самозаверяющий сертификат при развертывании в лабораторной среде.|
    ![Изображение конфигурации центра ATA](media/ATA-Center-Configuration.JPG)

7.  Щелкните **Установить**, чтобы установить ATA и его компоненты, а также установить подключение между центром ATA и консолью ATA.

8.  По завершении установки нажмите кнопку **Запустить**, чтобы подключиться к консоли ATA.

    Во время установки центра ATA устанавливаются и настраиваются следующие компоненты:

    -   IIS

    -   MongoDB

    -   служба центра ATA и сайт IIS консоли ATA;

    -   пользовательский набор сбора данных системного монитора;

    -   самозаверяющие сертификаты (если выбраны во время установки).

> [!NOTE]
> Чтобы улучшить продукт и устранить неполадки, рекомендуется установить MongoVue и любую другую надстройку MongoDB или средства стороннего производителя по своему усмотрению. Для MongoVue необходимо установить платформу .NET Framework 3.5.

### Проверка установки

1.  Убедитесь, что служба центра Microsoft Advanced Threat Analytics запущена.

2.  На рабочем столе щелкните ярлык Microsoft Advanced Threat Analytics, чтобы подключиться к консоли ATA. Войдите под теми же учетными данными пользователя, которые использовалась для установки центра ATA. Во время первого входа в консоль ATA вы автоматически перейдете на страницу **Параметры подключения к домену**. Там вы сможете продолжить настройку и развертывание шлюзов ATA.

3.  Проверьте файл ошибок в файле **Microsoft.Tri.Center-Errors.log**, который по умолчанию расположен по следующему пути: %programfiles%\Microsoft Advanced Threat Analytics\Center\Logs.
